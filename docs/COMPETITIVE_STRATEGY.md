# Media Assistant 竞品分析与破局策略

> **核心结论**：市面上在“免费 + 高性能 + 生产力血缘”这个细分领域，目前没有直接竞品。我们的策略是：**“免费基座 (70% Eagle 体验) + 独家杀手锏 (血缘/非侵入式)”**。

## 1. 市场格局现状 (Market Landscape)

| 维度 | Eagle (付费霸主) | Billfish (免费竞品) | Media Assistant (我们) |
| :--- | :--- | :--- | :--- |
| **模式** | 买断制 (¥199) | 免费 (部分增值) | **免费开源基座 + 插件生态** |
| **定位** | 素材管理 (存图/看图) | 素材管理 | **创作者的第二大脑 (生产力/协作)** |
| **优势** | 体验极致，格式全，插件强 | 免费，功能较全 | **非侵入式，资产血缘，工程解析** |
| **劣势** | 侵入式(改文件名)，无血缘 | **性能差(10w+卡顿)**，数据私有 | 格式支持暂缺，预览体验待提升 |

---

## 2. 我们的核心定位

**不要试图做一个更好的 Eagle，要做一个 Eagle 做不了的 Media Assistant。**

我们不卷“素材库”（单纯的存图），我们要卷的是**“工作流” (Workflow)**：
1.  **非侵入式 (Non-intrusive)**：
    *   **对手**：把文件“吃”进私有库，改变路径，导致 PR/AE 无法链接。
    *   **我们**：**只做索引，不动文件**。用户文件保留在原处，不仅是素材库，更是文件系统的增强层。
2.  **资产血缘 (Asset Lineage)**：
    *   **痛点**：一张图不知道是从哪个 PSD 导出的；一个视频不知道用了哪些素材。
    *   **我们**：独家提供 **“来源 -> 产物”** 的双向追踪，解决“文件关系”问题。
3.  **工程解析 (Project Parsing)**：
    *   **痛点**：移动文件后，PR/剪映 工程红名（丢失链接）。
    *   **我们**：通过指纹索引，自动帮用户**修复工程链接**。

## 3. 具体竞争策略

### Phase 1: 夯实基座 (对标 Eagle)
*目标：达到 Eagle 70%-80% 的核心体验，让用户“可用”。*

*   **格式大补全**：
    *   [x] 图片/视频 (基础支持)
    *   [ ] **3D 模型** (FBX, OBJ, BLEND) - *高优先级*
    *   [ ] **设计源文件** (PSD, AI, SKETCH) - *高优先级*
    *   [ ] **专业格式** (RAW, EXR, TGA, HDR)
    *   [ ] **字体 & 音频**
*   **视觉检索**：
    *   [x] **颜色搜索**：按主色调找图 (已实现算法与API)。
    *   [ ] **形状/以图搜图** (未来规划)。

### Phase 2: 降维打击 (对标 Billfish)
*目标：在免费市场中，用“高性能”和“开放性”碾压对手。*

*   **超大库性能优化**：
    *   **痛点**：Billfish 在 10 万+ 文件时会显著卡顿。
    *   **解法**：Rust/Go 高性能后端 + 前端虚拟列表 (Virtual List) + 缩略图预加载。
    *   **指标**：百万级文件秒开，内存占用低。
*   **智能集合 (Smart Collections)**：
    *   **功能**：自动分类（如“所有大于100MB的红色视频”）。
    *   **状态**：后端模型与API已设计，待实装。
*   **数据主权**：
    *   支持将 Metadata 写入 Sidecar 文件或 XMP，用户数据不被软件锁死，随时可迁移。

## 4. 总结

**市面上还有能打的吗？**
- **没有了**。
- **Inboard / Pixave**：已停止维护或太小众。
- **Playbook / Air Inc**：云端订阅制，面向企业，不是个人创作者的直接竞品。

**我们是重度创作者（视频/游戏/开发）的唯一解**。只要稳住“看图/存图”的基本盘，配合“血缘”和“非侵入”的独家功能，我们将在生态位上拥有不可替代性。
